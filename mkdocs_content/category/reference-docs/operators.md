# Operators
Learn how operators control workflow execution using built-in constructs such as branching, conditions, and loops in Orkes Conductor.

Use operators when a workflow needs control flow such as branching, loops, parallelism, waiting, sub-workflows, or termination.

Operators define orchestration structure while workers and system tasks execute the business logic at each step.

## Recommended path

If you are unsure where to begin, start with **Switch**, **Do While**, **Wait**, **Dynamic**, **Set Variable**. These pages cover the most common entry points for this section.

## When to use this section

Use these pages to answer implementation questions, compare available primitives, and find the exact guide or reference page for the next step in your Orkes Conductor project.
