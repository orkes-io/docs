# Message Broker Integrations
Configure Kafka, AMQP, NATS, SQS, Azure Service Bus, GCP Pub/Sub, IBM MQ, and managed Kafka providers for event-driven workflows in Orkes Conductor.

Use these integrations when workflows need event-driven triggers, async messaging, or external queue coordination.

Connect brokers such as Kafka, Confluent, SQS, Azure Service Bus, Pub/Sub, NATS, AMQP, or IBM MQ for reliable event orchestration.

## Recommended path

If you are unsure where to begin, start with **Apache Kafka**, **Confluent Kafka**, **Amazon MSK**, **AMQP / RabbitMQ**, **NATS Messaging**. These pages cover the most common entry points for this section.

## When to use this section

Use these pages to answer implementation questions, compare available primitives, and find the exact guide or reference page for the next step in your Orkes Conductor project.
